## 一、产品概述

### 1.1 产品定位
"基金投资大师"是一款面向个人投资者的基金查询与深度投研工具，用户输入基金名称或代码后，可实时查询基金当前价格并获取 AI 生成的深度投研报告。

### 1.2 目标用户
- 基金投资者（新手/资深）
- 需要快速了解基金基本面的用户
- 希望获得专业投研观点的个人投资者

### 1.3 核心价值
- ✅ **快速查询**：支持基金名称/代码模糊搜索
- ✅ **实时数据**：对接盈米基金 MCP 获取最新净值
- ✅ **深度报告**：AI 生成专业投研分析（媲美机构研报）
- ✅ **移动友好**：H5 网页版，无需下载，即开即用

## 二、功能需求

### 2.1 核心功能清单

| 优先级 | 功能模块 | 功能描述 |
|--------|----------|----------|
| P0 | 基金搜索 | 支持基金名称、代码、拼音缩写搜索 |
| P0 | 基金详情 | 展示基金基本信息、实时净值、涨跌幅 |
| P0 | AI 投研报告 | 生成深度投研分析报告（6 大维度） |
| P1 | 报告分享 | 支持分享报告到微信/朋友圈 |
| P1 | 历史查询 | 记录用户查询历史（本地存储） |
| P2 | 基金对比 | 支持 2-3 只基金对比分析 |
| P2 | 自选列表 | 用户可收藏关注的基金 |

### 2.2 功能详细说明

#### 2.2.1 基金搜索 (P0)
**输入方式**：
- 基金代码：6 位数字（如：000001、110022）
- 基金名称：完整或部分名称（如：易方达、华夏成长）
- 拼音缩写：首字母（如：YFD、HXCZ）

**搜索结果**：
- 精确匹配：直接展示基金详情
- 模糊匹配：展示匹配列表供用户选择
- 无结果：提示"未找到相关基金"

#### 2.2.2 基金详情页 (P0)
**展示字段**：
| 字段 | 说明 | 来源 |
|------|------|------|
| 基金代码 | 6 位标准代码 | 盈米 MCP |
| 基金名称 | 完整名称 | 盈米 MCP |
| 单位净值 | 最新净值 | 盈米 MCP |
| 日涨跌额 | 较前一交易日涨跌 | 盈米 MCP |
| 日涨跌幅 | 涨跌百分比 | 盈米 MCP |
| 累计净值 | 历史累计净值 | 盈米 MCP |
| 基金类型 | 股票型/债券型/混合型等 | 盈米 MCP |
| 成立日期 | 基金成立时间 | 盈米 MCP |
| 基金规模 | 当前资产规模 | 盈米 MCP |
| 基金经理 | 现任经理姓名 | 盈米 MCP |

#### 2.2.3 AI 投研报告 (P0)
**报告结构**（6 大维度）：

```markdown
## 一句话投资判断
[用 1 句话概括基金的投资价值]

## 基金概况与定位
- 基金类型与投资策略
- 业绩比较基准
- 风险收益特征

## 历史业绩分析
- 近 1 月/3 月/6 月/1 年/3 年收益率
- 同类排名百分位
- 最大回撤与恢复时间

## 持仓分析
- 前十大重仓股/债券
- 行业分布
- 集中度分析

## 基金经理评估
- 经理履历与投资风格
- 管理该基金时长
- 历史管理业绩

## 风险评估
- 波动率分析
- 下行风险
- 适合的投资人群

## 投资建议
- 买入/持有/观望建议
- 适合的投资期限
- 配置建议
```

**生成方式**：
- 调用 AI 模型（qwen3.5-plus）
- 基于盈米 MCP 提供的基金数据
- 流式输出，用户可实时查看生成进度

#### 2.2.4 报告分享 (P1)
**分享渠道**：
- 微信好友
- 微信朋友圈
- 复制链接

**分享内容**：
- 基金名称 + 一句话投资判断
- 报告链接（H5 页面）

#### 2.2.5 查询历史 (P1)
**存储方式**：localStorage 本地存储
**展示内容**：
- 最近查询的 10 只基金
- 点击可快速查看详情
**清除功能**：一键清空历史

## 三、技术架构

### 3.1 技术栈选型

| 层级 | 技术 | 说明 |

| 前端框架 | Taro 4.x + React 18 | 跨端框架，支持 H5/小程序 |

### 3.3 核心 API 设计

#### 3.3.1 基金信息查询
```typescript
// 请求
POST /functions/v1/fund-info
Body: { code: string }  // 基金代码或名称

// 响应
{
  success: boolean,
  data?: {
    fund_code: string,      // 基金代码
    fund_name: string,      // 基金名称
    net_value: number,      // 单位净值
    nav_date: string,       // 净值日期
    daily_growth: number,   // 日涨跌额
    daily_growth_rate: number, // 日涨跌幅
    accumulated_nav: number, // 累计净值
    fund_type: string,      // 基金类型
   成立_date: string,       // 成立日期
    fund_scale: number,     // 基金规模 (亿元)
    fund_manager: string    // 基金经理
  },
  error?: string,
  multiple?: boolean,       // 是否多个匹配
  matches?: Array<{fund_code: string, fund_name: string}>
}
```

#### 3.3.2 AI 投研报告生成
```typescript
// 请求
POST /functions/v1/fund-analysis
Body: {
  fundInfo: {
    fund_code: string,
    fund_name: string,
    net_value: number,
    daily_growth_rate: number,
    fund_type: string,
    fund_scale: number,
    fund_manager: string
  }
}

// 响应 (流式 SSE)
data: {"choices":[{"delta":{"content":"..."}}]}
data: [DONE]
```

## 五、数据源对接

### 5.1 盈米基金 MCP

**MCP 服务端点**：待配置

**核心接口**：
1. `fund/search` - 基金搜索
2. `fund/detail` - 基金详情
3. `fund/nav` - 净值查询
4. `fund/holding` - 持仓信息
5. `fund/manager` - 基金经理
6. `fund/performance` - 历史业绩

### 5.2 数据缓存策略
- 基金基本信息：缓存 24 小时
- 净值数据：缓存至下一交易日
- AI 报告：缓存 7 天（相同基金+日期）

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 盈米 MCP 接口不稳定 | 数据获取失败 | 实现降级方案，缓存历史数据 |
