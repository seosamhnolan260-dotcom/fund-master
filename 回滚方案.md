## 🚨 回滚触发条件

### 严重级别定义

| 级别 | 定义 | 响应时间 | 决策权限 |
|------|------|---------|---------|
| 🔴 P0 - 严重 | 核心功能不可用 | 5 分钟内 | 自动回滚 |
| 🟠 P1 - 高危 | 主要功能异常 | 15 分钟内 | DevOps 决定 |
| 🟡 P2 - 中等 | 部分功能异常 | 30 分钟内 | 团队讨论 |
| 🟢 P3 - 轻微 | 非关键问题 | 1 小时内 | 修复优先 |

### 回滚触发场景

#### 🔴 P0 - 严重（立即回滚）

- [ ] Edge Functions 错误率 > 50%
- [ ] 前端页面完全无法加载
- [ ] 数据库连接失败
- [ ] 安全漏洞发现
- [ ] 数据泄露风险
- [ ] 核心功能（搜索/查询）完全不可用

#### 🟠 P1 - 高危（快速回滚）

- [ ] Edge Functions 错误率 > 20%
- [ ] AI 报告生成功率 > 80%
- [ ] 前端主要功能异常
- [ ] 性能严重下降（响应时间 > 10 秒）
- [ ] 移动端完全无法使用

#### 🟡 P2 - 中等（评估后回滚）

- [ ] Edge Functions 错误率 > 10%
- [ ] 部分基金查询失败
- [ ] AI 报告生成失败率 > 50%
- [ ] 性能下降（响应时间 > 5 秒）
- [ ] 特定浏览器兼容性问题

#### 🟢 P3 - 轻微（修复优先）

- [ ] UI 显示问题
- [ ] 非核心功能异常
- [ ] 文案错误
- [ ] 轻微性能问题

## 🔄 回滚流程

### 回滚决策流程

```
问题发现 → 严重性评估 → 回滚决策 → 执行回滚 → 验证确认 → 问题复盘
   ↓           ↓           ↓           ↓           ↓          ↓
 5 分钟      5 分钟      立即       10 分钟      10 分钟    24 小时内
```

### 回滚通知流程

1. **发现人** → DevOps（立即）
2. **DevOps** → 技术负责人（5 分钟内）
3. **技术负责人** → PM（15 分钟内）
4. **PM** → 相关干系人（30 分钟内）

## 🛠️ 回滚操作指南

### Edge Functions 回滚

#### 方法一：重新部署稳定版本（推荐）

```bash
# 1. 切换到稳定版本代码
cd /Users/jun15prolan/.openclaw/workspace/projects/基金投资大师
git checkout <stable-tag>  # 如 v0.9.0

# 2. 进入 supabase 目录
cd supabase

# 3. 重新部署所有函数
supabase functions deploy

# 4. 验证部署
supabase functions list

# 5. 测试函数
curl -X POST https://<project-ref>.supabase.co/functions/v1/fund-info \
  -H "Content-Type: application/json" \
  -d '{"code":"000001"}'
```

#### 方法二：单个函数回滚

```bash
# 回滚特定函数
cd supabase/functions

# 从备份恢复代码
cp ../functions-backup/<function-name>/index.ts <function-name>/index.ts

# 重新部署
supabase functions deploy <function-name>
```

#### 方法三：Supabase Dashboard 回滚

1. 登录 Supabase Dashboard
2. 进入 Edge Functions 页面
3. 选择要回滚的函数
4. 查看部署历史
5. 选择之前的版本
6. 点击 "Restore" 或 "Redeploy"

### 前端回滚

#### Vercel 回滚

```bash
# 方法一：CLI 回滚
vercel rollback <deployment-url>

# 方法二：指定部署
vercel --prod <deployment-url>

# 方法三：Dashboard 回滚
# 1. 登录 Vercel Dashboard
# 2. 选择项目
# 3. 查看 Deployments 历史
# 4. 选择稳定版本
# 5. 点击 "Promote to Production"
```

#### Netlify 回滚

```bash
# 方法一：CLI 回滚
netlify deploy --prod --dir=dist/build/h5 --message "Rollback to stable"

# 方法二：Dashboard 回滚
# 1. 登录 Netlify Dashboard
# 2. 选择项目
# 3. 查看 Deploys 历史
# 4. 选择稳定版本
# 5. 点击 "Publish deploy"
```

#### 手动回滚

```bash
# 1. 切换到稳定版本代码
cd /Users/jun15prolan/.openclaw/workspace/projects/基金投资大师
git checkout <stable-tag>

# 2. 重新安装依赖
pnpm install

# 3. 重新构建
pnpm build:h5

# 4. 重新部署
vercel --prod
# 或
netlify deploy --prod --dir=dist/build/h5
```

### 环境变量回滚

```bash
# 1. 查看当前环境变量
supabase secrets list

# 2. 记录当前值（用于问题排查）
supabase secrets list > env-backup-$(date +%Y%m%d-%H%M%S).txt

# 3. 恢复之前的环境变量
supabase secrets set YINGMI_MCP_ENDPOINT=<previous-value>
supabase secrets set YINGMI_MCP_API_KEY=<previous-value>
supabase secrets set QWEN_API_KEY=<previous-value>

# 4. 重新部署函数使环境变量生效
supabase functions deploy

# 5. 验证
supabase secrets list
```

### 数据库回滚

```bash
# 查看迁移历史
supabase db remote commit

# 回滚到特定迁移（谨慎操作）
supabase db reset --db-url <connection-string>

# 或手动执行回滚 SQL
# 在 Supabase Dashboard SQL Editor 中执行
```

**⚠️ 注意**: 数据库回滚需谨慎，可能导致数据丢失。建议先备份数据。

## 🔧 应急工具

### 快速回滚脚本

```bash
#!/bin/bash
# rollback.sh - 快速回滚脚本

set -e

STABLE_VERSION=${1:-v0.9.0}
PROJECT_DIR="/Users/jun15prolan/.openclaw/workspace/projects/基金投资大师"

echo "🔄 开始回滚到版本：$STABLE_VERSION"

# 1. 切换代码版本
cd $PROJECT_DIR
git checkout $STABLE_VERSION

# 2. 回滚 Edge Functions
cd supabase
echo "📦 回滚 Edge Functions..."
supabase functions deploy

# 3. 回滚前端
cd $PROJECT_DIR
echo "🏗️ 回滚前端..."
pnpm install
pnpm build:h5
vercel --prod

# 4. 验证
echo "✅ 验证回滚..."
curl -X POST https://<project-ref>.supabase.co/functions/v1/fund-info \
  -H "Content-Type: application/json" \
  -d '{"code":"000001"}'

echo "🎉 回滚完成！"
```
