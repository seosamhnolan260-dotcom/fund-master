## 🗄️ Supabase 部署

### 2.1 登录 Supabase

```bash
# 登录 Supabase (如未登录)
supabase login

# 验证登录状态
supabase whoami
```

### 2.2 链接项目

```bash
# 进入 supabase 目录
cd supabase

# 链接到生产项目
supabase link --project-ref <your-project-ref>

# 验证链接状态
supabase status
```

**需要替换**:
- `<your-project-ref>`: 从 Supabase Dashboard 获取的项目引用 ID

### 2.3 数据库迁移

```bash
# 执行数据库迁移
supabase db push

# 验证迁移结果
supabase db diff

# 或查看迁移历史
supabase db remote commit
```

### 2.4 配置环境变量

```bash
# 查看当前环境变量
supabase secrets list

# 设置盈米 MCP 端点 (待 PM 提供)
supabase secrets set YINGMI_MCP_ENDPOINT=https://mcp.yingmi.com/api

# 设置盈米 MCP API Key (待 PM 提供)
supabase secrets set YINGMI_MCP_API_KEY=<api_key>

# 设置阿里云百炼 API Key
supabase secrets set QWEN_API_KEY=<qwen_api_key>

# 设置 Supabase URL
supabase secrets set SUPABASE_URL=https://<project-ref>.supabase.co

# 设置 Supabase Service Role Key
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=<service_role_key>

# 设置缓存 TTL (可选，使用默认值)
supabase secrets set CACHE_FUND_INFO_TTL=86400
supabase secrets set CACHE_NAV_TTL=43200
supabase secrets set CACHE_REPORT_TTL=604800

# 再次验证所有环境变量
supabase secrets list
```

### 2.5 部署 Edge Functions

```bash
# 方式一：部署所有函数（推荐）
supabase functions deploy

# 方式二：逐个部署函数
supabase functions deploy fund-search
supabase functions deploy fund-info
supabase functions deploy fund-nav
supabase functions deploy fund-detail
supabase functions deploy fund-analysis
supabase functions deploy fund-holding
supabase functions deploy fund-performance

# 验证部署结果
supabase functions list
```

### 2.6 验证函数部署

```bash
# 查看函数日志
supabase functions logs --watch

# 测试函数调用（示例）
curl -X POST https://<project-ref>.supabase.co/functions/v1/fund-search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <anon-key>" \
  -d '{"query":"000001"}'
```

## 🌐 前端部署

### 3.1 配置前端环境变量

```bash
# 回到项目根目录
cd /Users/jun15prolan/.openclaw/workspace/projects/基金投资大师

# 创建生产环境配置
cp .env.example .env.production

# 编辑 .env.production
# 填入以下配置：
# TARO_APP_SUPABASE_URL=https://<project-ref>.supabase.co
# TARO_APP_SUPABASE_ANON_KEY=<anon-key>
# TARO_APP_APP_ID=fund-investment-master
```

### 3.2 代码检查

```bash
# 运行代码检查
pnpm run lint

# 检查输出是否有错误
# 如有错误，修复后重新运行
```

### 3.3 生产构建

```bash
# 执行生产构建
pnpm build:h5

# 验证构建输出
ls -la dist/build/h5

# 检查构建产物大小
du -sh dist/build/h5
```

### 3.4 部署到 Vercel

```bash
# 登录 Vercel
vercel login

# 首次部署（交互式）
vercel

# 生产部署
vercel --prod

# 或指定目录部署
vercel --prod --cwd dist/build/h5
```

### 3.5 部署到 Netlify（备选）

```bash
# 登录 Netlify
netlify login

# 首次部署（交互式）
netlify deploy

# 生产部署
netlify deploy --prod --dir=dist/build/h5
```

### 3.6 配置自定义域名

```bash
# Vercel 配置域名
vercel domains add fund-master.example.com

# Netlify 配置域名
# 在 Netlify Dashboard 中配置
```

## 🔧 故障排查

### 常见问题及解决方案

#### 1. Supabase CLI 登录失败

```bash
# 清除缓存重新登录
rm -rf ~/.supabase
supabase login
```

#### 2. 函数部署失败

```bash
# 查看详细错误
supabase functions deploy --debug

# 检查函数代码
cat supabase/functions/<function-name>/index.ts
```

#### 3. 前端构建失败

```bash
# 清理缓存
rm -rf node_modules dist
pnpm install
pnpm build:h5
```

#### 4. 环境变量未生效

```bash
# 重新设置环境变量
supabase secrets set KEY=value --replace

# 重新部署函数
supabase functions deploy <function-name>
```
