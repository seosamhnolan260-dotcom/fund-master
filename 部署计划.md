## 🔄 四、回滚方案

### 4.1 回滚触发条件

| 情况 | 严重级别 | 响应时间 | 回滚决策 |
|------|---------|---------|---------|
| Edge Functions 500 错误率 > 10% | 🔴 严重 | 5 分钟 | 立即回滚 |
| 前端页面无法加载 | 🔴 严重 | 5 分钟 | 立即回滚 |
| AI 报告生成失败率 > 50% | 🟡 中等 | 15 分钟 | 评估后决定 |
| 基金查询响应 > 10 秒 | 🟡 中等 | 15 分钟 | 评估后决定 |
| 非关键功能异常 | 🟢 轻微 | 1 小时 | 修复优先 |

### 4.2 Edge Functions 回滚

```bash
# 1. 查看当前部署版本
supabase functions list

# 2. 回滚到上一版本（如支持）
supabase functions deploy <function-name> --version <previous-version>

# 3. 或重新部署已知稳定版本
cd /path/to/stable/code
supabase functions deploy
```

### 4.3 前端回滚

```bash
# Vercel 回滚
vercel rollback <deployment-url>

# Netlify 回滚
# 在 Netlify Dashboard 中选择之前的部署并点击 "Publish deploy"

# 或重新部署稳定版本
cd /path/to/stable/code
pnpm install
pnpm build:h5
vercel --prod
```

### 4.4 环境变量回滚

```bash
# 查看当前环境变量
supabase secrets list

# 恢复之前的环境变量值
supabase secrets set <KEY>=<previous-value>
```

## 🚀 五、部署步骤

### 5.1 阶段一：环境准备 (30 分钟)

```bash
# 1. 确认 Supabase CLI 安装
supabase --version

# 2. 登录 Supabase
supabase login

# 3. 链接项目
cd /Users/jun15prolan/.openclaw/workspace/projects/基金投资大师/supabase
supabase link --project-ref <your-project-ref>

# 4. 验证连接
supabase status
```

### 5.2 阶段二：数据库迁移 (10 分钟)

```bash
# 执行数据库迁移
supabase db push

# 验证表创建
supabase db diff
```

### 5.3 阶段三：环境变量配置 (20 分钟)

```bash
# 设置盈米 MCP 配置 (待 PM 提供)
supabase secrets set YINGMI_MCP_ENDPOINT=<endpoint>
supabase secrets set YINGMI_MCP_API_KEY=<api_key>

# 设置阿里云百炼配置
supabase secrets set QWEN_API_KEY=<qwen_api_key>

# 设置 Supabase 配置
supabase secrets set SUPABASE_URL=<supabase_url>
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=<service_role_key>

# 验证环境变量
supabase secrets list
```

### 5.4 阶段四：Edge Functions 部署 (30 分钟)

```bash
# 部署所有函数
supabase functions deploy

# 或逐个部署
supabase functions deploy fund-search
supabase functions deploy fund-info
supabase functions deploy fund-nav
supabase functions deploy fund-detail
supabase functions deploy fund-analysis
supabase functions deploy fund-holding
supabase functions deploy fund-performance

# 验证部署
supabase functions list
```

### 5.5 阶段五：前端构建部署 (30 分钟)

```bash
# 1. 进入项目目录
cd /Users/jun15prolan/.openclaw/workspace/projects/基金投资大师

# 2. 安装依赖
pnpm install

# 3. 代码检查
pnpm run lint

# 4. 配置生产环境变量
cp .env.example .env.production
# 编辑 .env.production 填入生产配置

# 5. 生产构建
pnpm build:h5

# 6. 部署到 Vercel/Netlify
vercel --prod
# 或
netlify deploy --prod --dir=dist/build/h5
```

### 5.6 阶段六：联调测试 (60 分钟)

```bash
# 1. 测试基金搜索
curl -X POST https://<project-ref>.supabase.co/functions/v1/fund-search \
  -H "Content-Type: application/json" \
  -d '{"query":"000001"}'

# 2. 测试基金信息查询
curl -X POST https://<project-ref>.supabase.co/functions/v1/fund-info \
  -H "Content-Type: application/json" \
  -d '{"code":"000001"}'

# 3. 测试 AI 报告生成

  -H "Content-Type: application/json" \

# 4. 前端页面测试
# 访问部署后的 H5 页面进行完整功能测试
```
